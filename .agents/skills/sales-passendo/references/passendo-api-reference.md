# Passendo API Reference

## Overview

Passendo provides a REST API for programmatic access to campaign management and reporting. The API is hosted at `api.hsttrckr.com`.

**Important**: The API documentation is JS-rendered and was not fully accessible during research. The information below is compiled from support center references, integration pages, and review sites. Verify all details against the actual API docs (requires Passendo account).

## Authentication

- **Method**: Unknown — likely API key or OAuth token (verify with Passendo)
- **Base URL**: `https://api.hsttrckr.com`
- **Documentation**: Available to Passendo customers at the base URL

## Known capabilities

Based on platform documentation and reviews, the API supports:

- Campaign management (CRUD operations)
- Reporting and analytics (impressions, clicks, CTR, revenue)
- Creative/banner management
- Placement configuration

## Gaps

The following could not be determined from public sources:

- Exact endpoint paths and HTTP methods
- Request/response JSON formats
- Authentication method and token format
- Pagination pattern
- Rate limits
- Webhook support (none found in public docs)
- Error response format

## Workaround

For programmatic access without the API:
- All campaign management is available through the Passendo dashboard (UI)
- Contact Passendo support for API documentation access
- No Zapier/Make/MCP alternatives available

## Recommendation

If you need the API, request documentation during onboarding. Passendo provides dedicated human support for integration, so your account manager can provide API credentials and docs directly.
